# Plan: s04 — Hooks (absorbing s03 Permission)

## Strategy

Implement **s04 directly**, which naturally includes permission logic as one of its hooks.
No separate s03 module needed — the hook system makes permission an extension point,
not a hard-coded gate.

## Why skip s03

The original s03 adds a 3-gate permission pipeline hard-coded into the loop.
But s04 immediately refactors that into a hook (`permission_hook` registered on `PreToolUse`).
Implementing s04 in one step gives us both the hook architecture AND the permission system,
without the intermediate hard-coded version.

## What s04 delivers

| Event | Trigger point | Example hooks |
|-------|---------------|----------------|
| `UserPromptSubmit` | Before user query reaches LLM | `context_inject_hook` — log/annotate input |
| `PreToolUse` | Before each tool execution | `permission_hook` (deny list + user approval), `log_hook` |
| `PostToolUse` | After each tool execution | `large_output_hook` — warn on oversized results |
| `Stop` | When loop exits (no more tool calls) | `summary_hook` — session stats |

## Implementation

### New module: `src/pendula/hooks.py`

```
HOOKS registry — dict of event → list[callable]
register_hook(event, callback)
trigger_hooks(event, *args) → str | None
```

### Changes to existing modules

| Module | Change |
|--------|--------|
| `agent.py` | Replace hard-coded permission check with `trigger_hooks("PreToolUse", block)`. Add `trigger_hooks("PostToolUse", block, output)` after execution. Add `trigger_hooks("Stop", messages)` when loop ends. |
| `repl.py` | Add `trigger_hooks("UserPromptSubmit", query)` before appending to history. |
| `tools.py` | No changes needed. |

### Permission hooks (built-in, registered at import time)

- `deny_list_hook` — hard deny patterns (rm -rf /, sudo, shutdown, etc.)
- `destructive_hook` — ask user for potentially destructive commands
- `write_outside_hook` — ask user for writes outside workspace
- `log_hook` — log every tool call
